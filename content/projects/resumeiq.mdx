## The Problem

Manual resume screening is slow, inconsistent, and prone to bias. HR teams spend hours parsing PDFs only to miss qualified candidates due to keyword mismatches or formatting differences.

**ResumeIQ** was built to solve exactly that.

## What It Does

ResumeIQ is a full-stack AI-powered platform that:

- **Parses** uploaded PDF/DOCX resumes and extracts structured data
- **Scores** candidates against job descriptions using GPT-4o
- **Optimizes** resumes with actionable suggestions to improve ATS pass rates
- **Ranks** applicant pools so recruiters focus on the right candidates first

## Technical Architecture

### Frontend
Built with **Next.js 16** using the App Router. The UI is a clean SaaS dashboard with Tailwind CSS and Framer Motion animations. Server Components handle the data fetching while Client Components manage interactivity.

### AI Layer
The core AI functionality is powered by **OpenAI GPT-4o** with structured output mode. Prompts are carefully engineered to return consistent JSON schemas for scoring and recommendations. I implemented a streaming response pattern so users see results appear in real-time.

### Data Layer
**Supabase** (PostgreSQL) stores all resume metadata, scores, and job descriptions. File uploads go directly to **Supabase Storage** with signed URLs for security.

### Processing Pipeline

```
Upload PDF → Extract Text → GPT-4o Analysis → Score + Recommendations → Store in DB → Render Results
```

## Key Challenges

**1. PDF Text Extraction** — PDFs with complex layouts or image-based content often produce garbled text. I implemented a fallback chain: native text extraction → OCR via Tesseract → manual review flag.

**2. Token Management** — Long resumes can exceed GPT context limits. I built a chunking and summarization pipeline to handle documents of any length while preserving important details.

**3. Consistent AI Output** — Getting GPT to always return valid, typed JSON required extensive prompt iteration and a JSON schema validation layer with Zod.

## Results

- Resume analysis time reduced from **45 minutes** to **under 30 seconds**
- ATS optimization suggestions increased interview callbacks in beta testing by **~40%**
- Platform handled **500+ resume analyses** in the first month of testing

## Lessons Learned

Building AI-powered products requires treating the LLM as an unreliable external service — with retries, fallbacks, and rigorous output validation. The engineering challenge isn't calling the API; it's making the whole system reliable at scale.
